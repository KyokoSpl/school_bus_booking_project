## 8. Protokollierung der Einwilligungen

### 8.1 Zu protokollierende Informationen

Für jede erteilte Einwilligung wird folgendes protokolliert:

| Feld | Beschreibung |
|------|--------------|
| Einwilligungs-ID | Eindeutige Kennung |
| Betroffene Person | Benutzer-ID oder E-Mail |
| Art der Einwilligung | z.B. Newsletter, Cookies |
| Zeitpunkt | Datum und Uhrzeit |
| IP-Adresse | Zur Verifizierung |
| Einwilligungstext-Version | Versionsnummer des verwendeten Textes |
| Status | Erteilt / Widerrufen |
| Widerrufsdatum | Falls widerrufen |

### 8.2 Datenbank-Schema

```sql
CREATE TABLE consent_log (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    consent_type VARCHAR(50) NOT NULL,
    consent_text_version VARCHAR(10) NOT NULL,
    granted_at TIMESTAMP NOT NULL,
    revoked_at TIMESTAMP,
    ip_address VARCHAR(45),
    user_agent TEXT,
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 9. Checkliste für rechtkonforme Einwilligungen

### 9.1 Anforderungen nach DSGVO

Jede Einwilligung muss folgende Kriterien erfüllen:

- [ ] **Freiwillig:** Keine Nachteile bei Verweigerung (außer bei notwendiger Verarbeitung)
- [ ] **Informiert:** Klare Information über Zweck, Daten, Empfänger, Dauer
- [ ] **Eindeutig:** Aktive Handlung (z.B. Checkbox ankreuzen, KEIN Pre-Checked)
- [ ] **Dokumentiert:** Nachweis der Einwilligung gespeichert
- [ ] **Widerrufbar:** Einfacher Widerruf, so einfach wie die Erteilung
- [ ] **Trennbar:** Separate Einwilligungen für verschiedene Zwecke
- [ ] **Verständlich:** Klare, einfache Sprache

### 9.2 Verbotene Praktiken

- ❌ Vorangekreuzte Checkboxen
- ❌ Kopplung an Vertragsabschluss (nur bei zwingender Notwendigkeit)
- ❌ Versteckte Einwilligungen in AGB
- ❌ Unklare oder irreführende Formulierungen
- ❌ Erschwerte Widerrufsmöglichkeit
